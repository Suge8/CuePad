你是资深桌面端产品工程师 + 前端架构师 + 世界级 ui 设计师 人机交互大师
我要做一个极轻量的本地桌面应用：给 coding agents 使用的提示词草稿本：CuePad

目标不是做 Notion，也不是复杂 Prompt Manager，而是：

- 打开快
- 写得爽
- 自动保存
- 一键复制
- 多段提示词拆开发送方便
- UI 精致、未来感
- Windows / macOS 都能用

## 技术栈

使用：

- Electron 37
- SvelteKit 最新稳定版
- Svelte 5
- TypeScript
- Tailwind CSS
- Bits UI
- Lucide Icons
- CodeMirror 6
- SQLite 本地存储

## 核心产品结构

应用有三层：
1. 项目
2. 卡片
3. 卡片正文

### 项目

项目类似分类。

每个项目包含：

- 名字
- 颜色
- icon
- 排序
- 创建时间
- 更新时间

例子：

- Viby
- Blog
- Claude Code
- Refactor

### 卡片
卡片是一整篇提示词草稿，不是一小段 snippet。
每个卡片包含：

- 标题
- 正文
- 所属项目
- 标签
- 收藏状态
- 排序
- 创建时间
- 更新时间

点击卡片后，进入全屏编辑状态。

## 编辑器要求

使用 CodeMirror 6。

不要做 Markdown 预览。  
正文就是纯文本 / Markdown-ish 编辑状态。

但可以做轻量视觉增强：

- `## 标题` 行看起来更醒目，但不要隐藏 `##`
- `- 列表` 行缩进更舒服，但不要隐藏 `-`
- 代码块用轻微背景突出
- `---split---` 显示成更明显的分段线，但原文仍然存在
- `{{变量}}` 可以轻微高亮，但不做变量系统

原则：

复制出去的内容必须等于用户实际写下的原始文本。

## 必做功能

第一版 MVP 只做：

1. 创建 / 编辑 / 删除项目
2. 项目可设置名字、颜色（必选）、icon、都是可选、都不选则留空
3. 创建 / 编辑 / 删除卡片
4. 卡片全屏编辑
5. 自动保存
6. 一键复制当前卡片全文
7. 支持 `---split---` 分段
8. 可复制全文 / 第 1 段 / 第 2 段 / 第 N 段
9. 全局搜索项目和卡片
10. 本地 SQLite 存储
11. 极简设置页

## 快捷键

- `Cmd/Ctrl + F`：打开搜索 / 命令面板
- `Esc`：退出全屏编辑
- `Alt/Option + Space（可设置）`：全局呼出应用小窗

其他快捷键先不要做。

## UI 风格

参考气质：

- Linear
- Raycast
- Arc
- Warp

视觉要求：

- 支持亮暗主题优先
- 低饱和背景
- 大圆角
- 轻阴影
- 动效丝滑明显
- 编辑区大留白
- 复制反馈明显
- 不要后台管理系统风格
- 不要模板感
- 不要复杂侧栏
- 字体和交互组件稍大 有设计感 字体使用'/Users/sugeh/Downloads/字体/OPPO Sans 4.0.ttf'

基础布局：

- 左侧：项目列表
- 中间：卡片列表
- 全屏：卡片编辑器

卡片进入编辑后沉浸，减少干扰。

## 自动保存

要求：

- 用户输入后自动保存
- 使用 debounce
- 显示保存状态：
  - 正在保存
  - 已保存
  - 保存失败
- 保存失败不能丢内容
- 应有本地错误恢复策略

## 数据模型

请设计 SQLite schema，至少包含：

- projects
- cards
- tags
- card_tags
- app_settings

需要考虑：

- 排序字段
- 创建时间
- 更新时间
- 软删除是否必要
- 后续版本历史扩展
